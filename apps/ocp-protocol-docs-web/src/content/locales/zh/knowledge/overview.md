# OCP 中文文档

> Open Commerce Protocol（OCP）是一组让 AI Agent、应用与异构商业系统之间能够互相发现商业对象、查询信息、解析详情、并触发动作的协议规范。

## OCP 一句话介绍

OCP 把"商业能力"抽象成一组可声明、可验证、可解释、可约束的协议面，让发现与交互独立于具体的商家、平台或后端实现——同一个 Agent 用同一种方式，就能在不同的电商、服务平台、招聘系统、企业 ERP 之间穿梭。

## OCP 解决什么问题

商业世界里能"被买、被预约、被申请、被联系"的对象散落在大量异构系统中：商品在不同电商平台，服务在本地预约平台，职位在 ATS，采购需求 / 渠道机会 / 报价 / 工作流入口又各属于不同企业的 CRM / ERP。AI Agent 想代表用户在这些系统之间穿梭，会同时遇到三类困难：

1. **找不到入口**——每家自有 API，没有统一发现机制。
2. **看不懂语义**——字段命名、可见性、新鲜度、来源都不一致。
3. **不敢下手**——没有可验证的权限、信任、审计基础，Agent 容易越权、发明字段或在用户没确认时下单。

OCP 在这些异构系统之上铺一层**协议语义**：让 Agent 用同一种方式发现合适的目录、查询对象、解析结果、并按受控的方式连接到下一步动作。
