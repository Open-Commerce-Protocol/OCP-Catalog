# OCP Documentation

> Open Commerce Protocol (OCP) is a set of protocols that lets AI agents, applications, and heterogeneous commerce systems discover, query, resolve, and act on commercial objects — without each agent having to learn every merchant's private API.

## OCP in one sentence

OCP models commerce capability as a layer of declarable, verifiable, explainable, and constrainable protocol surfaces. The same agent can move across storefronts, service platforms, recruiting systems, and enterprise ERPs using one shared semantic — without being locked into any specific merchant, platform, or backend.

## What OCP addresses

Things you can "buy, book, apply to, or contact" live in many heterogeneous systems: products in e-commerce platforms, services in booking platforms, jobs in ATSes, RFQs / opportunities / quotes / workflow entry points across CRMs and ERPs. An AI agent acting on behalf of a user across these systems hits three recurring problems:

1. **No discovery layer** — every platform has its own API.
2. **No shared semantics** — field names, visibility, freshness, and provenance vary widely.
3. **No trust foundation** — without verifiable permissions, provenance, and audit, agents tend to invent fields, exceed authorization, or place orders without explicit user confirmation.

OCP places a protocol layer over these systems so agents use one consistent approach to find catalogs, query objects, resolve results, and connect to actions in a controlled way.

## Who this is for

- Engineers, architects, and protocol enthusiasts working on **Agentic Commerce**.
- Teams building **agent-driven shopping, services, or recruiting flows**.
- E-commerce and SaaS developers connecting existing merchant systems to AI agents.
- Payments practitioners designing **trusted agent and payment-instruction flows**.
