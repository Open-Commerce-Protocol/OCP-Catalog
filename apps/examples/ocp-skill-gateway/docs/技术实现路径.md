# OCP Skill Gateway — 技术实现路径

> 把 OCP Catalog 包装成 LLM / Agent 平台可直接调用的 Skill,
> 同时为商家 / 学生开发者侧提供数据看板。
>
> 目标平台共 **3 类、9+ 家**:
>
> - **国内 OpenAPI 派**:Coze(扣子)、腾讯元器、阿里百炼、文心智能体、Dify
> - **海外 OpenAPI 派**:ChatGPT Custom GPT Actions、OpenAI Assistants API
> - **MCP 派**:Claude Desktop、claude.ai 集成、Cursor、Cody、Codex CLI
> - **本地 Skill 包**:Claude Code `.skill`
>
> 关键洞察:**一份 OpenAPI 3.0 服务 7 家,一个 MCP server 服务 5 家**,
> 真正需要单独打包的只有 Claude Code Skill 一种。

---

## 1. 项目定位

```
        OpenAPI 派                    MCP 派                    Skill 包
   Coze / 元器 / 百炼              Claude Desktop             Claude Code
   文心 / Dify / ChatGPT  ──┐      Cursor / Cody  ──┐         .skill 文件 ──┐
   OpenAI Assistants        │     Codex CLI         │                       │
                            ▼                      ▼                       ▼
                       ┌────────────────────────────────────────────────────┐
                       │           OCP Skill Gateway                        │
                       │  /skill/*  (HTTP+OpenAPI)   ←── ocp-mcp-server     │
                       │  /dashboard/*  (浏览器)          (转译 MCP→HTTP)   │
                       │  /admin/*                                          │
                       └────────────────────────────────────────────────────┘
                                  │
                                  ▼
                       OCP Catalog Nodes  (alimama / JD / PDD / ...)
                                  │
                                  ▼
                            telemetry 落地
                          (调用量 / 转化 / 佣金)
```

**三个职责放在同一个包**:

| 职责 | 路由前缀 | 服务对象 |
|---|---|---|
| Skill API | `/skill/*` | LLM Agent(经平台插件转发) |
| Dashboard API | `/dashboard/*` | 浏览器(学生 / 运营 / 商家) |
| Admin API | `/admin/*` | 内部(API Key 管理、catalog 注册) |

**MCP 复用**:[apps/ocp-mcp-server/](../../../ocp-mcp-server/) 已经存在,
Week 1 末把它的 tools 转成 fan-out 到 `/skill/*` 即可,Claude / Cursor / Codex 全覆盖。

## 2. 5 个 MVP Skill

| Skill | Endpoint | OCP 底层 | 优先级 |
|---|---|---|---|
| 搜商品 | `POST /skill/search` | broker.search 跨 catalog fan-out | P0 |
| 生成购买链接 | `POST /skill/deeplink` | broker.resolve 返回带返佣的 URL | P0 |
| 比价 | `POST /skill/compare` | search 同 SKU/关键词 → 按价聚合 | P1 |
| 按预算推荐 | `POST /skill/recommend` | search + 过滤排序 | P1 |
| 订单/佣金查询 | `POST /skill/order` | 各联盟订单回流 | P2 |

**设计原则(给 LLM 调的接口)**:

- 入参用自然语言友好字段:`query: string`、`budget_max: number`、`category: string`
- 出参扁平、无内部字段(不暴露 `entry_id`、`object_id` 等 OCP 内部 token)
- 每个 endpoint 的 `description` / `summary` 要像写给同事看,LLM 靠这个选工具

## 5. 关键架构决策

### 5.1 鉴权:API Key per 学生

- 学生通过管理后台领取 API Key
- Skill 平台插件配置 → Header: `X-Skill-Key: sk_xxx`
- telemetry 每条记录带 `api_key_id`,用于:
  - 限流 / 滥用控制
  - 分润归因(学生的 Agent 带来的订单可追溯)

### 5.2 telemetry 存储

- **M1**: 进程内 Map + 定期 flush 到 JSONL(零依赖,先跑通)
- **M2**: 切到 SQLite / PostgreSQL(沿用 `packages/db` 的 drizzle)
- 不要一上来就 Kafka / ClickHouse,先看数据量

### 5.3 OpenAPI Spec 生成

- 用 `@elysiajs/swagger` 或手写 YAML,选后者(各平台对 spec 有细微差异,手写更好控制)
- spec 只包含 `/skill/*`(平台审核的就是这部分),不暴露 `/dashboard` / `/admin`

### 5.4 跨 catalog fan-out

- Skill Gateway **不直接调**京东/淘宝/拼多多接口
- 调的是已经搭好的 OCP Catalog Node(`alimama-catalog-api`, `jdunion-catalog-api`, `pdd-catalog-api` 等)
- catalog 节点列表通过 env 配置(后续接入 Registration Center)

### 5.5 MCP 复用现有 server

- [apps/ocp-mcp-server/](../../../ocp-mcp-server/) 已有 `@modelcontextprotocol/sdk` 基础设施
- **不另起 MCP server**,而是改它的 tool 实现:把原本直连 catalog 的逻辑改为 HTTP fan-out 到 `/skill/*`
- 这样 Claude Desktop / Cursor / Codex CLI 只要安装这一个 MCP server,就能拿到所有 skill

### 5.6 Claude Code Skill 包

- Claude Code 的 skill 格式见仓库根 `skills-lock.json`
- 每个 skill 是带前置 `# Description` 的 Markdown + 配套 OpenAPI 引用
- Week 3 单独打 1-2 个示例 skill,验证学生开发者的"用 OCP 写 Agent"路径

## 7. 风险与未知项

| 风险 | 影响 | 缓解 |
|---|---|---|
| **平台对返佣/广告类工具的政策** | 可能审核拒绝或下架(尤其 OpenAI / Anthropic) | Week 2 提交前先读 marketplace 政策;准备 fallback 用纯"商品信息查询"形态 |
| **MCP 标准在演进** | Anthropic 改协议导致已发布配置失效 | 锁 `@modelcontextprotocol/sdk` 版本,关注 changelog |
| **学生分润法律合规** | 涉及给个人结算佣金,可能需要发票/代扣 | 业务/法务侧定型,M2 之前必须有结论 |
| **跨境联盟数据合规** | 海外平台审核时如果指向国内联盟,可能被卡 | Week 2 ChatGPT 上架时如遇阻,先做 AliExpress / 海外联盟 |
| **公网域名 + 备案** | 国内平台部分要求备案域名 | 业务侧准备一个备案过的子域,海外用云上未备案域 |

| 决策 | 影响 | 谁来定 |

| Skill Gateway 域名 / 部署位置(国内 + 海外) | 各平台审核需要 HTTPS 公网域名,可能要 2 个域 | @AW |
| 分润机制 | API Key 体系怎么落到结算;法务合规 | @AW |
| 学生开发者活动 | 时间 / 预算 / 招募范围 | @AW |
| 是否开放 B 端商家 | 决定 dashboard 是否要做"商家视角"切换 | @AW |
| 是否针对 Claude / Codex 用户群单独运营 | 影响 MCP server 是否纳入 OCP 官方推荐目录 | @AW |
