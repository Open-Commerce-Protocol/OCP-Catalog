# 公网暴露 — Cloudflare Quick Tunnel

> 用 `cloudflared` 把 `localhost:4330` 暴露成 `https://*.trycloudflare.com`,
> 给 Coze / ChatGPT / 元器 / 百炼 等平台的插件后台导入 OpenAPI 用。
>
> 不需要 Cloudflare 账号、不需要域名,**zero-config**。

## 一次性准备(只做一次)

```bash
winget install --id Cloudflare.cloudflared --accept-package-agreements --accept-source-agreements --silent
cloudflared --version   # 期望:cloudflared version 2026.x.x
```

## 启动公网链路(每次开发)

四个终端,或者用 `start-all.ps1` 之类的脚本组织。

```bash
# 1) JD Union mock (:4320)
bun run --cwd apps/examples/jdunion-catalog-api start

# 2) PDD mock (:4340)
PDD_CATALOG_PORT=4340 bun run --cwd apps/examples/pdd-catalog-api start

# 3) Alimama mock (:4311)
#    注意:Alimama 默认 :4310,但 QQ 等本地软件可能占该端口
ALIMAMA_CATALOG_PORT=4311 bun run --cwd apps/examples/alimama-catalog-api start

# 4) Skill Gateway (:4330)
#    注意:SKILL_GATEWAY_PUBLIC_BASE_URL 必须先填一个占位,等 tunnel 起来再覆盖
env \
  SKILL_GATEWAY_API_KEYS=sk_dev_demo_001 \
  SKILL_GATEWAY_PUBLIC_BASE_URL=http://localhost:4330 \
  SKILL_GATEWAY_CATALOGS='[{"id":"cat_jdunion","name":"JD Union","base_url":"http://localhost:4320"},{"id":"cat_pdd","name":"PDD","base_url":"http://localhost:4340"},{"id":"cat_alimama","name":"Alimama","base_url":"http://localhost:4311"}]' \
  bun apps/examples/ocp-skill-gateway/src/index.ts

# 5) Cloudflare Quick Tunnel
cloudflared tunnel --url http://localhost:4330 --no-autoupdate
# 输出里会有一行:
#   https://<random-words>.trycloudflare.com
# 复制这个 URL
```

## 让 OpenAPI 的 servers.url 指向公网

平台审核时会以 `openapi.yaml` 里的 `servers.url` 为准。
拿到 tunnel URL 后,**重启 gateway 把 `SKILL_GATEWAY_PUBLIC_BASE_URL` 改成它**:

```bash
env \
  SKILL_GATEWAY_API_KEYS=sk_dev_demo_001 \
  SKILL_GATEWAY_PUBLIC_BASE_URL=https://<random-words>.trycloudflare.com \
  SKILL_GATEWAY_CATALOGS='[{...}]' \
  bun apps/examples/ocp-skill-gateway/src/index.ts
```

验证:

```bash
curl -s https://<random-words>.trycloudflare.com/openapi.yaml | grep -A2 'servers:'
# 期望:
#   servers:
#     - url: https://<random-words>.trycloudflare.com
```

## 全链路回归

```bash
SKILL_GATEWAY_SMOKE_BASE=https://<random-words>.trycloudflare.com \
SKILL_GATEWAY_SMOKE_KEY=sk_dev_demo_001 \
bun run smoke:skill-gateway
# 期望:result: 13/13 passed
```

## 已知坑

| 坑 | 表现 | 解法 |
|---|---|---|
| Quick tunnel URL 每次启动都变 | tunnel 重启 → 平台后台已配置的 URL 失效 | 测试阶段够用;真要上架前升级到 [named tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/) + 自己的域名 |
| Alimama 默认 :4310 被 QQ 占 | curl :4310 返回乱码而不是 JSON | 加 `ALIMAMA_CATALOG_PORT=4311` |
| PDD 默认 :4330 和 gateway 撞车 | gateway 起不来或 PDD 起不来 | 加 `PDD_CATALOG_PORT=4340` |
| Windows 下 bash kill 杀不掉 bun 子进程 | `kill <pid>` 无效,端口仍被占 | `netstat -ano \| grep :PORT` → `powershell Stop-Process -Id <pid> -Force` |

## 长期方案(Week 3+)

quick tunnel 用来跑 demo 没问题,真要上线学生开发者活动:

1. 业务侧准备一个**备案过的子域**(国内平台审核要求)+ 一个**未备案的子域**(海外平台)
2. cloudflared 改用 [named tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/),绑定上面两个域
3. 上一个 systemd / nssm 让 tunnel 跑成服务,机器重启自动拉回来
4. 流量监控接 Cloudflare Analytics
