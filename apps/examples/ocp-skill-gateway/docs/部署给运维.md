# OCP Skill Gateway — 部署给运维 / 领导一页说明

> 一句话:把 OCP 注册中心(`ocp.deeplumen.io/mcp`)的 catalog 能力暴露成 LLM Agent 平台
> (Coze / 元器 / 百炼 / ChatGPT GPT Actions)能识别的 HTTP OpenAPI 插件。

## 为什么需要部署

目前 gateway 还跑在开发者本地 + cloudflared quick tunnel(`*.trycloudflare.com`),URL 每次重启
就漂、没有 SLA、个人电脑关机就挂。要给 Coze 等平台正式上架,需要:

- 一个**稳定的 HTTPS 公网入口**
- **24×7 在线**
- 跟 `ocp.deeplumen.io` 同一组域名信誉(强烈建议挂在 `*.deeplumen.io` 子域名下)

## 推荐部署形态

- **域名**:`skill-gateway.deeplumen.io`(或者 `ocp.deeplumen.io/skill-gateway` 子路径)
- **容器**:`ocp-skill-gateway:0.1.0`,单进程,无状态,无数据库依赖
- **端口**:容器内 4330(HTTP),前面建议挂 nginx / cloudflare 做 TLS 终结 + 限流
- **资源**:128MB 内存 + 0.1 vCPU 起步够用(实测 idle 30MB)

## 它依赖什么

唯一依赖一个出网调用:**`https://ocp.deeplumen.io/mcp`**(OCP 注册中心 MCP server)。
其它什么都不要 —— 没有数据库,没有 Redis,没有第三方账号(alimama / 京东等都在注册中心后端封装好了)。

## 怎么 build

构建上下文 = 仓库根目录(`OCP-Catalog/`),Dockerfile 在 `apps/examples/ocp-skill-gateway/Dockerfile`。

```bash
# 在仓库根目录跑
docker build \
  -f apps/examples/ocp-skill-gateway/Dockerfile \
  -t ocp-skill-gateway:0.1.0 \
  .
```

镜像约 80~100 MB(`oven/bun:1.3-alpine` 基础 + 单文件 bundle 1.3 MB)。

## 怎么 run

最小启动:

```bash
docker run -d --name skill-gateway \
  -p 4330:4330 \
  -e SKILL_GATEWAY_API_KEYS=sk_prod_xxx,sk_prod_yyy \
  -e SKILL_GATEWAY_PUBLIC_BASE_URL=https://skill-gateway.deeplumen.io \
  ocp-skill-gateway:0.1.0
```

或者用 `docker compose`:

```bash
SKILL_GATEWAY_API_KEYS=sk_prod_xxx \
SKILL_GATEWAY_PUBLIC_BASE_URL=https://skill-gateway.deeplumen.io \
  docker compose -f apps/examples/ocp-skill-gateway/docker-compose.yml up -d
```

## 必填 env

| 变量 | 必填 | 示例 | 说明 |
|---|---|---|---|
| `SKILL_GATEWAY_API_KEYS` | ✅ | `sk_prod_a,sk_prod_b` | 给 Coze / 元器 等平台用的 X-Skill-Key,逗号分隔多个,无则启动会 panic |
| `SKILL_GATEWAY_PUBLIC_BASE_URL` | ✅ | `https://skill-gateway.deeplumen.io` | 写进 `/openapi.yaml` 的 `servers.url`,LLM 平台靠这个回调 |

## 可选 env(都有默认值)

| 变量 | 默认 | 说明 |
|---|---|---|
| `SKILL_GATEWAY_PORT` | `4330` | 容器内监听端口 |
| `SKILL_GATEWAY_UPSTREAM` | `ocp_mcp` | 上游模式;`local_catalogs` 切回本地 mock(开发用) |
| `SKILL_GATEWAY_OCP_MCP_URL` | `https://ocp.deeplumen.io/mcp` | 仅 `ocp_mcp` 模式生效 |
| `SKILL_GATEWAY_FANOUT_TIMEOUT_MS` | `8000` | 上游 fan-out 单次调用超时 |
| `SKILL_GATEWAY_TELEMETRY_SINK` | `in-memory` | 调用流水落地;`jsonl` 写文件 |

## 健康检查

容器自带 healthcheck,30 秒 ping 一次 `/health`,docker ps 会显示 `healthy / unhealthy`。

人工验证:

```bash
curl https://skill-gateway.deeplumen.io/health
# {"ok":true,"service":"ocp-skill-gateway","upstream":"ocp_mcp",
#  "upstream_url":"https://ocp.deeplumen.io/mcp","api_keys_loaded":2}

curl -X POST https://skill-gateway.deeplumen.io/skill/search \
  -H 'content-type: application/json' \
  -H 'X-Skill-Key: sk_prod_xxx' \
  -d '{"query":"耳机","page_size":3}'
# 期望:total=6 左右,items[].source 含 "Alimama Affiliate Catalog"
```

## 反向代理 / TLS 建议

如果用 nginx 在前面:

```nginx
location /skill-gateway/ {
    proxy_pass http://127.0.0.1:4330/;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    # gateway 内部不处理 X-Forwarded-*,所以 SKILL_GATEWAY_PUBLIC_BASE_URL 必须显式设对
    proxy_read_timeout 30s;  # 上游 MCP 慢时给点余量
}
```

如果用 Cloudflare 前面挂域名:DNS A 记录 → 服务器 IP,Origin Server 配 `http://gateway-host:4330`,TLS 直接靠 Cloudflare,无需服务器侧 HTTPS。

## 路由清单(对外)

| 路径 | 谁调 | 鉴权 | 说明 |
|---|---|---|---|
| `POST /skill/search` | LLM 平台插件 | `X-Skill-Key` | 跨 catalog 搜商品(fan-out)|
| `POST /skill/deeplink` | 同上 | 同上 | 把搜索结果项 → 真返佣购买链接 |
| `POST /skill/compare` | 同上 | 同上 | 跨平台比价(按价升序)|
| `POST /skill/recommend` | 同上 | 同上 | 按预算 / 类目推荐 |
| `POST /skill/order` | 同上 | 同上 | M2 占位,返 501 |
| `GET /openapi.yaml` | 平台导入插件用 | 无 | 公网开放,给 Coze / 元器 等 |
| `GET /health` | 监控 | 无 | healthcheck |
| `GET /dashboard/*` | 浏览器(M1 无鉴权) | 无 | 给学生 / 运营看的看板 API |

注意 `/dashboard/*` 现阶段没有鉴权,部署到公网时**建议在 nginx 层把 `/dashboard/*` 屏蔽掉**或限定内网 IP。

## 滚动升级

镜像无状态,直接 `docker pull <新 tag> && docker compose up -d` 即可。
旧容器停止时未 inflight 完成的 LLM 请求会被中断,LLM 平台一般会自动重试。

## 出问题怎么查

```bash
docker logs ocp-skill-gateway --tail=200       # 启动日志 + 异常
docker exec ocp-skill-gateway wget -qO- localhost:4330/health   # 内部探活
docker exec ocp-skill-gateway wget -qO- localhost:4330/dashboard/catalogs   # 上游 catalog 列表
```

常见现象:

- 启动立刻退出 + zod 报错:`SKILL_GATEWAY_API_KEYS` 没设或为空 → 加 env 重启
- `/skill/search` 返回所有 catalog 都 `ok:false`:出网到 `ocp.deeplumen.io/mcp` 被防火墙拦了 → 加白名单
- `servers.url` 显示 `http://localhost:4330`:`SKILL_GATEWAY_PUBLIC_BASE_URL` 没设 → 加 env 重启
