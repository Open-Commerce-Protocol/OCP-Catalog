## 申请步骤（5 步、按顺序做）

### 步骤 1 — 注册多多进宝推广员账号

打开 **https://jinbao.pinduoduo.com** 用拼多多账号登录。
- 首次进入会要求实名认证（个人）或上传营业执照（公司）。
- 实名信息**与决策 1 选定的主体一致**。

> ⚠️ 提示：拼多多账号尽量用一个长期可控的账号,不要用临时账号或他人账号。
> 推广员账号一旦绑定就是永久关联。

### 步骤 2 — 拿到推广员 ID（member_id）

主体审核通过后,进入"我的账号"→"账号信息",会看到一个 **`推广员 ID`**（也叫 `member_id`,一串 10 位数字左右,如 `26829999`）。
这个数字在第 4 步组成 PID 时会用到。**不需要单独发给技术**,但请记下来。

### 步骤 3 — 注册 PDD 开放平台开发者 + 申请服务市场应用（拿 ClientId/ClientSecret）

打开 **https://open.pinduoduo.com** 登录。流程：

1. **成为开发者**（如果还不是）：完善公司/个人开发者信息
2. **进入"服务市场"或"应用管理"**
3. **创建应用**：
   - 应用类型：选**自用应用**（关键!）
   - 应用名称：`ocp-pdd-poc` 或类似
   - 应用描述：`AI Agent 导购 PoC，对接多多进宝 API 实现商品发现与转链`
   - 关联类目：选"多多进宝"或"营销推广"
   - 申请的 API 权限：勾选以下 4 个
     - ✅ `pdd.ddk.goods.search`（物料搜索）
     - ✅ `pdd.ddk.goods.detail`（单品详情）
     - ✅ `pdd.ddk.goods.promotion.url.generate`（转链）
     - ✅ `pdd.ddk.order.list.increment.get`（增量订单）
4. **绑定多多进宝账号**：选择步骤 2 注册好的推广员账号(同主体)
5. **提交审核**：等待 **1–3 工作日**

审核通过后,应用详情页能看到：
- **`ClientId`**（一串字母数字,约 16 位）
- **`ClientSecret`**（一串字母数字,约 32 位）

**这是第 1、2 个要给技术的东西。**

### 步骤 4 — 创建推广位（拿 PID）

回到 **https://jinbao.pinduoduo.com**,进入"推广管理"→"我的推广位"→"创建推广位"：

- 推广位名称：`ocp-poc-001`
- 推广场景：选"内容场景"或"导购场景"（任意能用的即可）
- 媒体类型：选"网站"或"自定义"
- 描述：`OCP Agent 推荐`

创建后会得到 **`PID`**,形如 **`26829999_278234567`**（前段是步骤 2 的推广员 ID,后段是位置 ID）。
**这是第 3 个要给技术的东西。**

> 💡 PoC 阶段只需要 **1 个 PID**。所有 Agent 共用这一个推广位,订单先归到统一账户。
> 后期想做"千 Agent 千归因"不需要更多 PID —— PDD 用 **`custom_parameters`** 字段透传 Agent ID 实现, 不依赖多 PID（这是 PDD 比 alimama / JD 更友好的地方）。

### 步骤 5 — 准备 1 个测试 goods_id

任选一个拼多多在售商品,从地址栏拿 `goods_id`。

形如：`https://mobile.yangkeduo.com/goods.html?goods_id=250012345678` → goods_id = `250012345678`

**优先选择**：
- ✅ "百亿补贴"内的商品（佣金率稳定、不易下架、口碑好）
- ✅ "多多农场"、"品牌馆"等长期在售类目
- ❌ 不要选限时秒杀 / 即将下架 / 拼团失败概率高的

**建议**：选一个生活日用品（耳机 / 充电器 / 牙膏 / 猫粮）,便于演示时不引起隐私争议。

## FAQ

**Q：审核会被拒吗？常见原因是什么？**
A：PDD 审核比京东严格。常见拒审原因：
1. 应用描述写得太敷衍（如只写"测试"） → 写清楚"AI 导购 Agent 推荐链路 PoC"+ 说明会接什么 API
2. 应用类型选了"推广引流类"但没有公开发布计划 → 改回"自用应用"
3. 个人主体申请 → 改成公司主体重提（**这是最常见的拒审原因**）
4. API 权限勾太多（如同时勾"商家管理"等高权限 API） → 只勾我们要的 4 个

**Q：要花钱吗？**
A：**全程 0 元**。多多进宝是 CPS 分润模式（用户成交后从商家佣金里抽 10% 服务费）,账号注册和 API 调用都免费。

**Q：调用 API 有频次限制吗？**
A：基础权限默认 **10 万次/天**（goods.search）/ **1 万次/天**（promotion.url.generate）,足够 PoC 演示。
高频场景需要在应用管理里申请提升,需要历史调用数据,PoC 阶段用不上。

**Q：佣金多久能到账？**
A：用户下单 → 确认收货 → 审核通过 → 结算（次月 15 号左右）→ 到账。
PoC 演示阶段拿不到结算佣金（演示时间太短）,但能看到"预估佣金"实时累计。

**Q：测试时下单后能取消订单吗？要不要真花钱？**
A：演示时可以"加购物车 → 不下单"或"下单 → 立刻取消",**不需要真实付款**。
联盟系统在用户下单瞬间就会记录归因,只要 PID 透传成功就能看到订单写入 ledger。

**Q：custom_parameters 是什么？要不要顺便申请？**
A：PDD 多多进宝支持 `custom_parameters` 字段（一个 JSON 字符串）,转链时透传,订单回流时原样回显。我们用它实现"千 Agent 千归因"——同一个 PID 下,不同 Agent 用不同 uid 标识,佣金能在账本里按 Agent 拆分。
**不需要单独申请**,custom_parameters 是 API 调用时随便传的字符串,账号本身就支持。这是 PDD 比京东（subUnionId 需要后台开权限）和阿里（adzone_id 兼任两个角色没法拆）更友好的地方。

**Q：PDD 与 JD 申请最大的差异？**
A：3 点：
1. **个人主体过审难度**：JD 个人主体能过,PDD 个人主体经常被卡——尽量用公司
2. **多入口**：PDD 需要同时操作 `jinbao.pinduoduo.com`（拿 PID）+ `open.pinduoduo.com`（拿 ClientId）,JD 只需要 `union.jd.com`
3. **审核时长**：PDD 服务市场应用审核 1–3 天,JD AppKey 通常当天

**Q：拿到 ClientId 后多久能看到效果？**
A：技术拿到凭据后,**当天就能跑通真实链路**：
1. 把 4 个环境变量填进 `.env`
2. `PDD_MOCK=false`
3. `bun run smoke:pdd`（mock smoke 改 1 行就变成真实 smoke）
4. 30 秒后能看到真 `p.pinduoduo.com/xxx` 短链,点击跳手 J 拼多多 App
