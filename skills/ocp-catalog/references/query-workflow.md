# Query Workflow

1. Discover or search a Registration Node.
2. Choose a Catalog from the returned route hints.
3. Inspect the Catalog manifest.
4. Pick a declared query pack. Do not use a query pack that is absent from the manifest.
5. Build the query from declared fields only.
6. Review returned entries and explanations.
7. Resolve only the entry selected by the user or required by the workflow.
8. Check freshness, policy, action binding expiry, and user confirmation requirements before any action.

Search is for candidate discovery. Resolve is for policy-aware detail and action entrypoints.
