# Code of Conduct

## Our standards

Open Commerce Protocol is a technical community for commerce protocol design,
reference implementations, integrations, and agent-facing tooling.

We expect contributors to:

- Be direct, respectful, and specific in technical discussion.
- Critique designs, patches, and tradeoffs rather than people.
- Assume protocol changes need evidence, clear compatibility notes, and review.
- Keep issues and pull requests focused on actionable outcomes.
- Respect maintainers' decisions when they close or redirect work.

Unacceptable behavior includes:

- Harassment, insults, threats, or sustained personal attacks.
- Discriminatory language or conduct.
- Publishing another person's private information without permission.
- Repeatedly derailing technical discussions after maintainers ask you to stop.
- Using security reports, private disclosures, or privileged access for harm.

## Scope

This code of conduct applies to all project spaces, including GitHub issues,
pull requests, discussions, review comments, community chats, and events where
someone is participating as part of the OCP Catalog project.

## Enforcement

Maintainers may remove comments, close threads, reject contributions, block
users, or take other proportionate action when behavior harms the project or
its contributors.

If you need to report a conduct issue, contact the maintainers privately. If a
private channel is not available, use the maintainer contact issue template to
ask for a private contact without including sensitive personal details.

Reports should include:

- What happened.
- Where it happened.
- Who was involved.
- Any links, screenshots, or timestamps that help maintainers verify the issue.

Maintainers will review reports as promptly as possible and will limit access
to report details to people who need them to respond.

## Attribution

This policy is adapted from common open source community conduct practices and
tailored for the OCP Catalog repository.
